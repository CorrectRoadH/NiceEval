Compare:

- **before**: [`examples/zh/origin/claude-sdk`](https://github.com/CorrectRoadH/niceeval/tree/main/examples/zh/origin/claude-sdk) - a standalone `@anthropic-ai/claude-agent-sdk` HTTP service.
- **after**: [`examples/zh/tier1/claude-sdk`](https://github.com/CorrectRoadH/niceeval/tree/main/examples/zh/tier1/claude-sdk) - the same app with NiceEval integration files.

**Integration style**: official converter plus stream driver. The adapter relays native `SDKMessage` frames, maps them through `fromClaudeSdkMessages`, uses `driveFrameStream` for ordered streaming, stores server-side session ids in `ctx.session`, and uses `pausable` state for HITL approval.

## File map

```text
claude-sdk/
├── niceeval.config.ts
├── agents/
│   └── claude-sdk.ts
├── evals/
│   ├── basic-qa.eval.ts
│   ├── hitl-approve.eval.ts
│   ├── hitl-deny.eval.ts
│   ├── session-isolation.eval.ts
│   └── weather-tool.eval.ts
└── experiments/
    └── assistant.ts
```

## What to inspect

- `agents/claude-sdk.ts`: transport plus `fromClaudeSdkMessages`.
- HITL evals: approval returns completed action results; rejection returns `status: "rejected"`.
- Session eval: verifies that a captured session id resumes one conversation and `t.newSession()` starts another.

## Run it

```bash
cd examples/zh/tier1/claude-sdk
pnpm install
pnpm dev
npx niceeval exp assistant
npx niceeval view
```
