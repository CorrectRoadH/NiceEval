NiceEval is an agent-native eval tool inspired by [eve](https://eve.dev). It is built for low-friction DX: most projects can install it, write the first eval, and start checking agent behavior in about 10 minutes.

It can evaluate Claude Code / Codex plugins, hooks, and Skills, and it can evaluate your own AI agent application directly. Whether your agent is built with AI SDK, LangGraph, Pi, or a custom loop, you connect it through an adapter and keep the same eval surface.

## What problem it solves

Tools such as Langfuse and Braintrust are stronger as tracing and monitoring systems. NiceEval focuses on the tighter loop: **write evals, run evals, inspect results**.

## Architecture

```text
   evals/*.eval.ts
        |
        v
   +---------------------+
   |     niceeval        |
   +---------------------+
        |
        | Agent adapter (official)
        v
   +------------------------------+
   |        Docker Sandbox         |
   |   +------------------------+  |
   |   | Codex / Claude Code /  |  |
   |   | apps needing isolation |  |
   |   +------------------------+  |
   +------------------------------+
```

```text
   evals/*.eval.ts
        |
        v
   +---------------------+
   |     niceeval        |
   +---------------------+
        |
        | Agent adapter (official or custom)
        v
   +------------------------------+
   |      Your AI Agent            |
   |  AI SDK · LangGraph · Pi ·    |
   |  custom agent loop            |
   +------------------------------+
```

## Two integration modes

**Sandbox mode** fits coding agents such as Codex and Claude Code that must edit files and run commands in a real workspace.

**Direct mode** fits your own AI agent application and does not require Docker.
