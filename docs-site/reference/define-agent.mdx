Every NiceEval agent is an adapter: code that knows how to drive a specific backend and translate output into the standard event stream. The runner only calls `agent.send(input, ctx)`.

![Full round trip of t.send: eval calls t.send, the runner builds TurnInput and ctx, the adapter returns a Turn with standard events.](/images/agent-turn-roundtrip-en.svg)

## `defineAgent`

Use this for direct-agent integrations:

```ts
import { defineAgent } from "niceeval/adapter";
```

<ParamField body="name" type="string" required>
  Unique agent name. Reports group results by this name.
</ParamField>

<ParamField body="capabilities" type="AgentCapabilities">
  Capabilities that cannot be proven automatically. Most adapters should leave this empty or declare only verified `conversation` / manual `toolObservability`.
</ParamField>

<ParamField body="events" type="EventSource">
  Optional external event source, usually `otelEvents()`, merged with events returned from `send`.
</ParamField>

<ParamField body="send" type="(input: TurnInput, ctx: AgentContext) => Promise<Turn>" required>
  Core function that drives the agent for one turn and returns a standard `Turn`.
</ParamField>

## `input`

<ResponseField name="text" type="string">
  Text sent by `t.send(...)`, file caption, or HITL answer text.
</ResponseField>

<ResponseField name="files" type="InputFile[] | undefined">
  Files attached by `t.sendFile(...)`, encoded as base64.
</ResponseField>

<ResponseField name="responses" type="InputResponse[] | undefined">
  Structured HITL answers from `t.respond(...)` / `t.respondAll(...)`.
</ResponseField>

<ResponseField name="outputSchema" type="JsonSchema | undefined">
  JSON Schema lowered from a `send` output declaration.
</ResponseField>

## `ctx`

<ResponseField name="signal" type="AbortSignal">
  Cancellation and timeout signal. Pass it to every request or child process you start.
</ResponseField>

<ResponseField name="session" type="AgentSession">
  State for the current conversation line: id capture, history storage, HITL hold/take, and a free `state` bag.
</ResponseField>

<ResponseField name="log(msg)" type="(msg: string) => void">
  Writes a turn log visible in `niceeval view`.
</ResponseField>

<ResponseField name="model" type="string | undefined">
  `experiment.model`, passed through unchanged.
</ResponseField>

<ResponseField name="flags" type="Readonly<Record<string, unknown>>">
  `experiment.flags`, passed through unchanged. Also available to evals as `t.flags`.
</ResponseField>

<ResponseField name="telemetry" type="Telemetry | undefined">
  Present when OTel integration is configured. Forward `telemetry.headers` to the app or `telemetry.env` to child processes.
</ResponseField>

<ResponseField name="sandbox" type="Sandbox | undefined">
  Present only for agents built with `defineSandboxAgent`.
</ResponseField>

## Direct-agent example

```ts
import { defineAgent } from "niceeval/adapter";

export default defineAgent({
  name: "web-agent",
  async send(input, ctx) {
    const r = await fetch("http://localhost:3000/chat", {
      method: "POST",
      headers: { "content-type": "application/json" },
      body: JSON.stringify({ message: input.text, flags: ctx.flags }),
      signal: ctx.signal,
    });
    const body = await r.json();
    return {
      status: r.ok ? "completed" : "failed",
      events: [{ type: "message", role: "assistant", text: body.reply }],
    };
  },
});
```

## `defineSandboxAgent`

Use this for coding agents that need a sandbox filesystem.

```ts
import { defineSandboxAgent } from "niceeval/adapter";
```

The adapter receives `ctx.sandbox`, which can upload files, run commands, read files, and collect workspace diffs. The selected sandbox backend comes from the experiment or project config.

```ts
export default defineSandboxAgent({
  name: "my-cli-agent",
  async send(input, ctx) {
    await ctx.sandbox.runCommand("my-agent", ["--prompt", input.text]);
    return {
      status: "completed",
      events: [{ type: "message", role: "assistant", text: "done" }],
    };
  },
});
```

## Registration

Experiments reference configured agent instances:

```ts
import { defineExperiment } from "niceeval";
import myBot from "../agents/my-bot.ts";

export default defineExperiment({
  agent: myBot,
});
```

## `ctx` and `t`

`ctx` is for adapter authors. It contains runner services and dynamic values for one turn.

`t` is for eval authors. It drives the agent and asserts facts.

They see the same run from opposite sides of the adapter boundary.
