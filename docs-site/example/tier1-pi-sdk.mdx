Compare:

- **before**: [`examples/zh/origin/pi-sdk`](https://github.com/CorrectRoadH/niceeval/tree/main/examples/zh/origin/pi-sdk) - a standalone `@earendil-works/pi-agent-core` HTTP service.
- **after**: [`examples/zh/tier1/pi-sdk`](https://github.com/CorrectRoadH/niceeval/tree/main/examples/zh/tier1/pi-sdk) - the same app with NiceEval integration files.

**Integration style**: manual transport with official stream helpers. The adapter maps native Pi frames through `fromPiAgentEvents`, uses `driveFrameStream` for ordered frame consumption, stores session ids in `ctx.session`, and pauses on approval requests for HITL.

## File map

```text
pi-sdk/
├── niceeval.config.ts
├── agents/
│   └── pi-sdk.ts
├── evals/
│   ├── basic-qa.eval.ts
│   ├── hitl-approve.eval.ts
│   ├── hitl-deny.eval.ts
│   ├── session-isolation.eval.ts
│   └── weather-tool.eval.ts
└── experiments/
    └── assistant.ts
```

## What to inspect

- `agents/pi-sdk.ts`: transport, `fromPiAgentEvents`, session capture, and pause/resume state.
- HITL evals: approval and rejection are separate regression paths.
- Weather eval: tool call assertions read standard `action.*` events.

## Run it

```bash
cd examples/zh/tier1/pi-sdk
pnpm install
pnpm dev
npx niceeval exp assistant
npx niceeval view
```
