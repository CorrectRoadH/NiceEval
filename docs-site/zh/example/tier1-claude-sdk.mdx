对比对象：

- **before**：[`examples/zh/origin/claude-sdk/`](https://github.com/CorrectRoadH/niceeval/tree/main/examples/zh/origin/claude-sdk) —— 一个独立的 `@anthropic-ai/claude-agent-sdk` HTTP 服务（`src/backend/server.ts`/`agent.ts`/`pending-approvals.ts`），还没接任何 eval。
- **after**：[`examples/zh/tier1/claude-sdk/`](https://github.com/CorrectRoadH/niceeval/tree/main/examples/zh/tier1/claude-sdk) —— 同一个应用接入 NiceEval 之后的样子。

应用侧的 `src/backend/*` **逐字节未变**——`agents/claude-sdk.ts` 把应用当黑盒，按需拉起它的
`server.ts` 子进程、轮询 `/healthz`，把原生 `SDKMessage` 流（`system` / `assistant` /
`user` / `result`）手写映射成标准事件流；`stream_event`（逐 token 渲染用）整个忽略。
这个应用没有 tracing（claude-code CLI 原生遥测只有 metrics+logs，niceeval 只消费
trace spans）。`calculate` 工具经 `canUseTool` 挂了 HITL 审批，被拒绝时靠
`system`/`permission_denied` 帧映射成 `status:"rejected"`。
