NiceEval helps you turn agent behavior into repeatable evals. It is designed for two common surfaces: coding agents that need a real workspace, and AI agent applications that already expose an HTTP, SDK, or framework interface.

## Architecture

### Mode 1: Sandbox

```text
   evals/*.eval.ts
        |
        v
   +---------------------+
   |     niceeval        |
   +---------------------+
        |
        | official sandbox agent
        v
   +------------------------------+
   |        Sandbox backend        |
   |   +------------------------+  |
   |   | Codex / Claude Code /  |  |
   |   | workspace apps         |  |
   |   +------------------------+  |
   +------------------------------+
```

Use this when the subject under test edits files, runs shell commands, or needs isolation. Evals prepare a workspace with `t.sandbox.uploadDirectory()`, then assert file changes, command output, diffs, and judge scores.

### Mode 2: Direct

```text
   evals/*.eval.ts
        |
        v
   +---------------------+
   |     niceeval        |
   +---------------------+
        |
        | adapter: defineAgent({ send })
        v
   +------------------------------+
   |      Your AI Agent            |
   |  AI SDK · LangGraph · Pi ·    |
   |  custom agent loop            |
   +------------------------------+
```

Use this when your app already has an interface. The adapter sends one turn to that interface and returns a standard `Turn`: status, events, usage, and optional structured data.

## Example

Running an eval takes two files: the eval itself, which says what to check, and an experiment, which says which agent and runtime configuration to use. The CLI does not accept a bare eval id as the thing to run. In `niceeval exp <experiment> <eval-prefix>`, the experiment decides which subject under test is connected.

```ts
import { defineEval } from "niceeval";
import { includes } from "niceeval/expect";

export default defineEval({
  description: "Weather questions should use the weather tool",
  async test(t) {
    await t.send("What's the weather like in Brooklyn today?");
    t.succeeded();
    t.calledTool("get_weather", { input: { city: "Brooklyn" }, count: 1 });
    t.check(t.reply, includes("sunny"));
  },
});
```

```bash
npx niceeval exp assistant weather
npx niceeval view
```

For a runnable direct-agent example, see `examples/zh/ai-sdk/`. For coding agents that need an isolated workspace, see `examples/zh/coding-agent-skill/`.
