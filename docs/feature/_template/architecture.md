# <功能名> —— 架构

这个功能内部怎么分层、和 core / Agent / Sandbox 的边界怎么划。

## 数据建模

先定实体关系，再定精确形状：

- 实体从属：谁包含谁、谁引用谁，画出层级（如 `Experiment ─ Attempt ─ Turn`）。
- 关联方式：实体之间靠什么关联（ID 引用、嵌套、事件顺序），生命周期由谁创建和销毁。
- 每个公开配置与结果类型给出穷尽形状——TS interface 代码块或字段表，写明字段语义与默认值。形状声明是穷尽的：未列出的字段即不存在。

## 数据流

输入到输出经过哪些阶段，每一步产出什么、由哪个模块负责。

## 不变量

无论实现怎么写都必须保持的性质。
