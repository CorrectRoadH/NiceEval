Compare:

- **before**: [`examples/zh/origin/langgraph`](https://github.com/CorrectRoadH/niceeval/tree/main/examples/zh/origin/langgraph) - a Python LangGraph / LangChain HTTP service.
- **after**: [`examples/zh/tier1/langgraph`](https://github.com/CorrectRoadH/niceeval/tree/main/examples/zh/tier1/langgraph) - the same app with TypeScript eval-side integration files.

**Integration style**: LangSmith OTel export plus adapter completion. LangSmith spans are consumed through `otelEvents({ dialects: [otel.langsmith] })`; where the dialect cannot recover message text or approval-gated tool details, the adapter supplements standard events from the app stream.

## File map

```text
langgraph/
├── niceeval.config.ts
├── agents/
│   └── langgraph.ts
├── evals/
│   ├── basic-qa.eval.ts
│   ├── hitl-approve.eval.ts
│   ├── hitl-deny.eval.ts
│   ├── session-isolation.eval.ts
│   └── weather-tool.eval.ts
└── experiments/
    └── assistant.ts
```

## What to inspect

- `agents/langgraph.ts`: how OTel-derived events and app-stream events are merged.
- `niceeval.config.ts`: fixed OTLP receiver port for the Python service.
- Session and HITL evals: the app protocol still owns these; spans do not replace them.

## Run it

```bash
cd examples/zh/tier1/langgraph
pnpm install
pnpm dev
npx niceeval exp assistant
npx niceeval view
```
